Git 命令速查表

Git 是目前最常用的分布式版本控制工具，适合代码托管、版本管理、多人协作开发等场景。

这篇文章不讲太多原理，主要整理日常开发中最常用的 Git 命令，方便快速查阅。

常用命令

场景 命令
查看状态 git status
简洁查看状态 git status -s
查看改动 git diff
查看已暂存改动 git diff --cached
添加全部文件 git add .
添加单个文件 git add 文件名
提交 git commit -m "说明"
查看提交记录 git log --oneline
拉代码 git pull
推代码 git push
获取远程更新 git fetch

分支命令

场景 命令
查看分支 git branch
查看所有分支 git branch -a
查看当前分支 git branch --show-current
新建并切换分支 git switch -c feature/demo
切换分支 git switch main
合并分支 git merge feature/demo
删除分支 git branch -d feature/demo
强制删除分支 git branch -D feature/demo

撤销命令

场景 命令
撤销工作区修改 git restore 文件名
取消暂存 git restore --staged 文件名
回退上一次提交，保留修改 git reset --soft HEAD~1
回退上一次提交，取消暂存 git reset --mixed HEAD~1
强制回退到指定版本 git reset --hard 提交ID
撤销某次提交 git revert 提交ID
查看操作记录 git reflog

stash 命令

场景 命令
暂存当前修改 git stash
查看暂存列表 git stash list
恢复最近一次 git stash pop
恢复指定 stash git stash apply stash@{0}
删除指定 stash git stash drop stash@{0}

远程仓库命令

场景 命令
查看远程仓库 git remote -v
添加远程仓库 git remote add origin 仓库地址
修改远程仓库地址 git remote set-url origin 新地址
首次推送分支 git push -u origin 分支名
拉取远程 main git pull origin main
获取远程 origin git fetch origin

危险命令大全

命令 危险点 常见用途
git reset --hard HEAD~1 直接丢弃最近提交后的修改 回退错误提交
git reset --hard 提交ID 工作区和暂存区一起回退，未保存修改会丢失 强制回到某个版本
git clean -fd 删除未跟踪文件和目录 清理本地脏文件
git clean -fdx 连 .gitignore 忽略文件也一起删除 彻底清空本地生成文件
git branch -D 分支名 强制删除未合并分支 删除废弃分支
git push --force 覆盖远程历史，可能影响他人 重写远程提交历史
git push --force-with-lease 仍然是强推，但比 --force 更安全 更新 rebase 后的远程分支
git rebase -i HEAD~N 会改写提交历史 整理提交记录
git commit --amend 修改最近一次提交哈希 改提交信息或补内容
git remote remove origin 删除远程仓库配置 重配远程地址

Tips:

看到 --hard、-D、clean、--force 这几个关键词时，执行前最好先确认一次。
